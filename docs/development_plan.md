# 《基于大语言模型的智能新闻摘要与协同互动系统》开发计划

## 一、总体开发思路

本项目采用“先公共框架、再模块并行、最后统一联调”的开发模式。

首先完成各服务的公共基础框架与统一约定，为小组成员提供稳定的协作基础；随后按业务模块并行开发，降低相互等待和代码冲突；最后集中完成接口、权限、数据与页面流程的集成联调，形成完整系统。

## 二、公共基础阶段

### 1. 第 0 阶段：项目总骨架搭建

创建前端、后端、AI 服务、文档、部署配置和开发脚本等一级目录，明确项目边界与协作分工。

### 2. 第 1 阶段：前端基础框架搭建

初始化前端工程，完成 Vue 3、路由、状态管理、请求封装、基础布局和公共组件等基础能力。

### 3. 第 2 阶段：后端基础框架搭建

初始化 FastAPI 后端工程，建立配置管理、接口分层、数据模型、统一响应和异常处理等基础结构。

### 4. 第 3 阶段：AI 服务框架搭建

搭建独立 AI 服务的接口与配置框架，先使用 Mock 返回结果，并为后续接入大语言模型 API 或本地模型服务预留扩展位置。

### 5. 第 4 阶段：用户与权限 Mock 搭建

建立用户、角色、登录状态与权限控制的 Mock 机制，为需要登录或分角色访问的页面、接口和流程提供统一支撑。

## 三、模块并行开发阶段

公共基础阶段完成后，以下模块可由不同小组成员或小组独立并行开发，并通过预先约定的接口与数据结构协作。

### 1. 新闻首页与详情模块

负责新闻列表、分类浏览、搜索入口、新闻详情展示及详情页相关交互。

### 2. AI 标题摘要生成模块

负责 AI 生成页面、请求参数组织、生成结果展示、结果使用与记录等功能。

### 3. 社区互动模块

负责社区内容浏览、发布、评论、点赞及其他互动流程。

### 4. 个人中心模块

负责个人资料、个人内容、生成记录、互动记录及个人设置等功能。

### 5. 审核/编辑与管理员后台模块

负责内容审核、编辑管理、用户管理、运营管理及后台统计等管理能力。

### 6. AI 服务模块

负责 AI 服务接口实现、Mock 数据维护、模型调用适配、提示词管理及生成结果处理。

## 四、集成联调阶段

各模块完成阶段性开发后，需要统一开展以下联调工作：

1. 前后端接口联调。
2. 后端与 AI 服务联调。
3. 页面跳转联调。
4. 权限联调。
5. Mock 数据统一。
6. 数据库接入替换。

联调期间应及时同步接口变更、字段约定和问题清单，避免模块间实现不一致。

## 五、开发优先级

### 第一优先级

项目骨架、前端框架、后端框架、AI 服务框架、用户权限 Mock、新闻首页、AI 生成页面。

### 第二优先级

社区、个人中心、互动功能。

### 第三优先级

审核后台、管理员后台、数据库接入、部署优化。

## 六、当前阶段说明

当前处于第 0 阶段，仅创建项目目录和开发文档，不初始化具体工程、不安装依赖，也不编写业务代码。
