# 系统部署说明文档

项目名称：智闻平台 / 基于大语言模型的智能新闻摘要与协同互动系统

本文档用于说明本项目在 Linux 服务器环境下的部署流程。文档参考《系统部署说明文档》模板的组织方式编写，并结合当前项目代码、README、配置文件、数据库脚本与启动命令整理。本文档中的服务器地址、密码、API Key 等敏感信息均使用占位符，实际部署时应按生产环境填写。

## 1. 部署环境说明

### 1.1 硬件环境

推荐部署在 Linux 云服务器或校内服务器，基础配置如下：

| 配置项 | 推荐配置 | 说明 |
| --- | --- | --- |
| CPU | 2 核及以上 | 满足 FastAPI 后端、AI 服务和 Nginx 基础运行 |
| 内存 | 4 GB 及以上 | 若启用真实大模型 API 调用，建议 8 GB 及以上 |
| 磁盘 | 40 GB 及以上 SSD | 用于存放项目代码、前端构建产物、日志、上传文件和数据库数据 |
| 网络环境 | 可访问公网或校园网 | 前端访问、接口访问、大模型 API 调用需要稳定网络 |

### 1.2 软件环境

| 软件 | 推荐版本 | 说明 |
| --- | --- | --- |
| 操作系统 | Ubuntu 20.04/22.04 或 CentOS 7/8 | 优先按 Linux 服务器部署 |
| Node.js | 18.x 及以上 | 用于构建 Vue3 + Vite 前端 |
| npm | 9.x 及以上 | 项目当前使用 npm 和 package-lock.json |
| Python | 3.10 及以上 | 用于运行 backend 和 ai-service |
| MySQL | 8.0 | 项目数据库 |
| Nginx | 1.18 及以上 | 提供前端静态资源和反向代理 |
| Git | 2.x | 代码拉取与版本管理 |
| 浏览器 | Chrome / Edge 最新版 | 系统访问与功能验证 |

### 1.3 部署辅助工具

- SSH 工具：XShell、FinalShell、MobaXterm 等。
- 文件上传工具：WinSCP、FileZilla、FinalShell 自带 SFTP。
- 数据库工具：Navicat、MySQL Workbench。
- 代码管理工具：Git。
- 命令行工具：Linux shell、PowerShell。

## 3. MySQL 数据库安装与初始化

### 3.1 MySQL 安装信息

本项目推荐使用 MySQL 8.0。数据库建议配置如下：

| 配置项 | 建议值 |
| --- | --- |
| 数据库名称 | `llm_news_system` 或根据实际部署环境填写 |
| 字符集 | `utf8mb4` |
| 排序规则 | `utf8mb4_unicode_ci` |
| 连接端口 | `3306` |

核心表包括：`user`、`news_category`、`news_topic`、`news`、`news_comment`、`user_like`、`favorite`、`browse_history`、`ai_generate_record`、`upload_file`、`community_post`、`post_comment`、`hot_topic`、`event_timeline`、`crawl_log`、`system_config` 等。

### 3.2 创建数据库

使用 root 或具备建库权限的 MySQL 用户登录后执行：

```sql
CREATE DATABASE IF NOT EXISTS llm_news_system
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

生产环境如需使用其他数据库名称，请将 `llm_news_system` 替换为实际数据库名。

### 3.3 创建数据库用户

如生产环境需要单独数据库用户，可执行以下 SQL。密码不要写入代码仓库或公开文档。

```sql
CREATE USER IF NOT EXISTS 'llm_news_user'@'localhost'
IDENTIFIED BY '<DB_PASSWORD>';

GRANT ALL PRIVILEGES ON llm_news_system.* TO 'llm_news_user'@'localhost';
FLUSH PRIVILEGES;
```

如果 backend 与 MySQL 不在同一台服务器，应根据实际网络策略调整 MySQL 用户 host，例如 `<BACKEND_SERVER_IP>`，并限制防火墙访问来源。

说明：展示数据库 `llm_news_system` 或生产环境实际数据库名，以及核心数据表。

### 3.4 导入数据库脚本

当前项目数据库脚本位于 `database/` 目录：

```text
database/schema.sql              # 初始表结构
database/seed.sql                # 基础演示数据
database/migrations/*.sql        # 增量迁移脚本
```

全新部署时，推荐先导入 `schema.sql`，再导入 `seed.sql`：

```bash
cd /opt/zhwen/llm-news-summary-collaboration-system
mysql --default-character-set=utf8mb4 -u llm_news_user -p llm_news_system < database/schema.sql
mysql --default-character-set=utf8mb4 -u llm_news_user -p llm_news_system < database/seed.sql
```

如果是在旧数据库上升级，应按 `database/migrations/` 中 SQL 文件编号顺序执行。当前迁移目录包含 `001`、`002`、`003`、`005`、`006`、`007`、`008`、`009`、`010`、`011`、`012`、`013`、`014`、`017` 至 `031` 等脚本，其中存在同编号不同用途脚本，应按文件名排序并结合实际数据库状态执行。

示例：

```bash
for file in database/migrations/*.sql; do
  echo "Running $file"
  mysql --default-character-set=utf8mb4 -u llm_news_user -p llm_news_system < "$file"
done
```

如生产环境对迁移顺序有严格要求，建议先在测试库执行并确认无报错后再应用到生产库。

## 7. Nginx 安装与配置

### 7.1 安装 Nginx

Ubuntu 示例：

```bash
sudo apt update
sudo apt install -y nginx
```

CentOS 示例：

```bash
sudo yum install -y nginx
```

### 7.2 配置前端静态资源

推荐通过 Nginx 提供前端静态资源，并将 `/api/` 请求反向代理到 backend。

示例配置：

```nginx
server {
    listen 80;
    server_name <SERVER_IP>;

    root /opt/zhwen/llm-news-summary-collaboration-system/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:8000/uploads/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

如果生产环境使用域名，应将 `server_name <SERVER_IP>;` 改为实际域名。

【截图 10：Nginx 配置截图】
说明：展示 `server`、`root`、`try_files`、`/api/ proxy_pass`、`/uploads/ proxy_pass` 等关键配置。

### 7.3 AI 服务是否需要代理

当前项目设计为前端只访问 backend，backend 再调用 `ai-service`。因此生产环境下 `ai-service` 不需要暴露公网端口，也不需要 Nginx 代理 `/ai/`。

推荐部署方式：

```text
浏览器 -> Nginx -> backend -> ai-service
```

只有在后续明确需要前端直接访问 AI 服务时，才需要增加 `/ai/` 反向代理：

```nginx
location /ai/ {
    proxy_pass http://127.0.0.1:8001/ai/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

sudo nginx -t
sudo systemctl start nginx
sudo systemctl enable nginx

### 7.5 防火墙端口

| 端口 | 服务 | 是否建议开放公网 |
| --- | --- | --- |
| 80 | HTTP 前端访问 | 是 |
| 443 | HTTPS 前端访问 | 如启用 HTTPS 则开放 |
| 8000 | backend | 建议仅本机或内网访问，由 Nginx 代理 |
| 8001 | ai-service | 不建议开放公网，仅 backend 内部调用 |
| 3306 | MySQL | 不建议开放公网 |

Ubuntu UFW 示例：

```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw reload
```

CentOS firewalld 示例：

```bash
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

## 8. 系统启动顺序

部署完成后建议按以下顺序启动：

1. 启动 MySQL。
2. 创建数据库并导入 `database/schema.sql`、`database/seed.sql`。
3. 配置并启动 `ai-service`。
4. 配置并启动 `backend`。
5. 配置前端环境变量并执行 `npm run build`。
6. 配置并启动或重载 Nginx。
7. 使用浏览器访问系统首页。

常用命令示例：

```bash
sudo systemctl start mysql

cd /opt/zhwen/llm-news-summary-collaboration-system/ai-service
source .venv/bin/activate
nohup uvicorn app.main:app --host 0.0.0.0 --port 8001 > ai-service.log 2>&1 &

cd /opt/zhwen/llm-news-summary-collaboration-system/backend
source .venv/bin/activate
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > backend.log 2>&1 &

sudo nginx -t
sudo nginx -s reload
```

## 10. 常见问题与处理方法

### 10.1 前端页面空白

处理方法：

- 检查 `frontend/dist` 是否打包成功并上传到 Nginx `root` 指定目录。
- 检查 Nginx `try_files $uri $uri/ /index.html;` 是否配置。
- 检查 `VITE_API_BASE_URL` 是否配置为生产后端地址。
- 打开浏览器开发者工具查看控制台错误。

### 10.2 接口 404

处理方法：

- 检查 Nginx `/api/` 反向代理是否配置正确。
- 检查 backend 是否启动在 8000 端口。
- 检查接口路径是否以 `/api` 为前缀。
- 访问 `http://<SERVER_IP>:8000/docs` 确认接口是否存在。

### 10.3 接口 500

处理方法：

- 检查 MySQL 是否启动。
- 检查 `backend/.env` 中数据库配置是否正确。
- 查看 `backend.log` 中的异常堆栈。
- 确认数据库表结构是否已导入最新 `schema.sql` 或 migrations。

### 10.4 AI 生成失败

处理方法：

- 检查 `ai-service` 是否启动。
- 检查后端 `AI_SERVICE_URL` 是否指向正确地址。
- 检查 `ai-service/.env` 中 `AI_MODE`、`SUMMARY_LLM_ENABLED`、`EVIDENCE_LLM_ENABLED` 配置。
- 若启用真实大模型，检查 `<LLM_API_KEY>`、`SUMMARY_LLM_BASE_URL`、`SUMMARY_LLM_MODEL` 是否正确。
- 如仅演示系统，可启用 Mock 模式。

### 10.5 数据库连接失败

处理方法：

- 检查 MySQL 服务状态。
- 检查数据库名、账号、密码是否正确。
- 检查 MySQL 用户 host 权限。
- 检查服务器防火墙和安全组。
- 使用命令手动测试连接：

```bash
mysql -h 127.0.0.1 -P 3306 -u llm_news_user -p llm_news_system
```

### 10.6 Nginx 配置不生效

处理方法：

```bash
sudo nginx -t
sudo nginx -s reload
sudo systemctl status nginx
sudo lsof -i:80
```

如 `nginx -t` 报错，应根据错误行号修改配置文件。
